Alive & Expressive

▶ Design rationale & motion notes

The concept. Eos is the dawn — so the status item is a tiny living organism made of a dawn-aurora gradient. Activity reads as life: it breathes, glows and spins faster the more agents are running. At a glance you feel the system's pulse without reading a word.

Idle is nearly invisible. A 6px desaturated grey dot on a slow 5.5s breath — present but quiet, so it never competes with the menu bar. No glow, no spin, no color.

Running is the signature. The dot blooms into a luminous orb with three layered motions: a breath (scale), a glow pulse (halo), and orbiting particles whose count and speed scale with the number of active agents — 1 agent gives a calm single orbit; many agents make a busy little constellation. A small count chip sits beside it. The gradient slowly hue-rotates so it's never static.

Completion is a queued celebration. Finishes never collide. Each one expands a compact pill from the orb — ✓ Name done with a spring-popped tick and a brief sparkle burst — holds ~1.4s, then collapses so the next queued finisher plays. After the queue drains it settles back to running (if work remains) or idle. This sequential playback is the interaction the whole concept is built around.

Menu-bar discipline. Compact by default (~22px); the pill caps its width and uses real spring easing rather than bounce-for-bounce's-sake. Vibrancy/blur on both light and dark, SF system type, tabular numerals. Motion is purposeful — every animation maps to a state, and prefers-reduced-motion would drop to opacity-only cross-fades.

The popover lists running + recently-completed agents with a live spinner dot, elapsed time and current activity; the just-finished row flashes mint. Clicking a row focuses that agent in Eos (simulated here with a highlight).
