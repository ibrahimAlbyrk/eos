Design rationale

◐ Glance over click

The bar item answers the two questions a power user asks most — how many are running and did anything just finish — without a click. A spinning dawn ring + tabular count + a breathing equalizer make "work is happening" legible in peripheral vision; the count uses tabular figures so the bar never reflows as it changes.

✓ Sequential completion queue

When several agents finish at once, completions queue and play one at a time — each name pops in with a green check, holds ~1.3s while a hairline drains, then the next slides up. A +N badge shows how many are still waiting, so a burst of 4 reads as a deliberate ticker, never a flicker. The bar settles back to the live view when the queue empties.

⌨ Dense, keyboard-first popover

One click opens a tight list: status glyph, monospaced name, live elapsed time, and a truncated activity line per agent — sorted running-first. ↑/↓ move a selection ring, ↵ focuses that agent in Eos, esc closes. Every pixel is information; nothing is decoration.

☾ Native restraint

Compact width, vibrancy blur, SF system type, and motion that is crisp and purposeful — the ring only spins while work runs, the equalizer stills at idle to near-invisibility. Tuned for both light and dark menu bars so it disappears into the OS until it has something to say.
