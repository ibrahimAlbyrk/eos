Design rationale & motion notes

The brief is the hardest kind: be invisible. A menu-bar utility earns trust by never demanding attention, so every decision here is subtractive.

Semantic colour only

Idle and running are strictly monochrome — the icon inherits the menu-bar tint (near-black on light, near-white on dark) exactly like every first-party item. Colour is reserved for meaning: system green for a completion, system red for a failure. Nothing is ever decorative.

Running stays dignified

“Living” is achieved with the smallest possible signal — a thin indeterminate arc and a quiet count, both in the bar tint. No pulsing glow, no brand hue. It reads as working, never as look at me.

The completion ticker

Completions queue and play as one continuous activity, not a burst of popups. The item reveals leftward (its right edge pinned, so neighbours and the popover never shift — true menu-bar behaviour), the checkmark strokes itself in, and between names the label does a subtle vertical swap while the width morphs. After the last one it collapses back to the icon. Sequential, never overlapping.

One easing, system materials

A single spring curve — cubic-bezier(.32,.72,0,1) — drives every transition, so width, fade and motion feel like one hand. Surfaces use real vibrancy (backdrop blur + saturation over the wallpaper), hairline borders and an inner top highlight, matching Control Center.
